# Анемон (AnemoneVPN) – Адаптивный протокол для обхода DPI

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Android](https://img.shields.io/badge/Android-Support-brightgreen)]()

**Анемон** — это экспериментальный VPN-протокол нового поколения, разработанный для противодействия современным системам глубокого анализа пакетов (DPI) и машинного обучения, используемым для блокировки трафика. Название выбрано не случайно: анемон (морской актиния) сливается со средой обитания – так и этот протокол маскирует трафик под легитимные приложения.

## Текущие возможности (версия 0.1.0)

Проект находится в активной разработке, но уже реализованы ключевые компоненты:

### ✅ Ядро протокола
- **TUN-интерфейс** – создание виртуального сетевого устройства для перехвата всего трафика устройства.
- **Криптографическое ядро**:
  - Шифрование AES-256-GCM с аутентификацией.
  - Обмен ключами ECDH (Perfect Forward Secrecy).
  - Автоматическая ротация сессионных ключей.
- **Фрагментация пакетов с TTL-маскировкой**:
  - Динамический размер фрагментов на основе статистических моделей реального трафика.
  - Генерация «умирающих» пакетов (с низким TTL) для запутывания анализа.
- **Подмешивание реального контента**:
  - Интеграция с публичными RSS-лентами, генерация случайных изображений для имитации легитимного трафика.
  - Перемешивание данных туннеля с реальным контентом.

### 🌐 Транспортный уровень
- **WebRTC транспорт** (на базе `aiortc`):
  - P2P-соединения с обходом NAT через ICE.
  - Сигнализация через WebSocket.
  - Data channel для передачи данных.
- **QUIC транспорт** (на базе `aioquic`):
  - Мультиплексирование потоков.
  - 0-RTT соединения.
  - Настраиваемые алгоритмы контроля перегрузки (Reno, Cubic, BBR).

### 🧠 Адаптивность и ML
- **ML-детектор** (на основе `scikit-learn` и `nfstream`):
  - Сбор статистических характеристик потока (размеры пакетов, интервалы, ретрансмиссии и т.д.).
  - Классификация трафика для выявления попыток DPI.
  - Обнаружение аномалий по правилам и машинному обучению.
- **Менеджер стратегий**:
  - Три уровня скрытности: Stealth, Balanced, Performance.
  - Пять профилей маскировки: Web, YouTube, Zoom, Telegram, Gaming.
  - Автоматическое переключение между стратегиями на основе ML-оценок и статистики успешности.

### 📊 Мониторинг и статистика
- Сбор метрик: отправленные/полученные байты, пакеты, RTT, число адаптаций.
- Периодический вывод статистики в лог.
- Callback-и для обработки событий подключения/отключения.

### 🖥️ CLI-интерфейс
- Запуск в режиме клиента или сервера.
- Выбор профиля, транспорта, уровня адаптивности.
- Поддержка конфигурационных файлов (YAML).

### 🔧 Поддерживаемые платформы
- **Linux** (полная поддержка TUN/TAP).
- **Android** (через отдельный клиент – см. раздел «Мобильная поддержка»).( в разработке)
- **macOS/Windows** – в планах (требуется адаптация TUN/TAP).

**Анемон** — это экспериментальный протокол VPN нового поколения, разработанный для противодействия современным методам глубокого анализа пакетов (DPI) и машинного обучения, используемым для блокировки трафика. Название выбрано не случайно: анемон (морской актиния) сливается со средой обитания, становясь незаметным — так же и этот протокол маскирует трафик под легитимные приложения.

## 🌟 Ключевые особенности

- **🔄 Динамическая фрагментация** - адаптивное разбиение пакетов на основе ML-моделей реального трафика
- **🎭 Имитация реальных протоколов** - трафик неотличим от YouTube, Zoom, Telegram и веб-серфинга
- **🛡️ TTL-маскировка** - использование "умирающих" пакетов для запутывания анализа
- **📦 Подмешивание реального контента** - внутри туннеля передаются настоящие новости и видео
- **🌐 Децентрализованная mesh-сеть** - на базе TON blockchain для устойчивости к блокировкам
- **🧠 Адаптивное поведение** - ML определяет попытки анализа и меняет стратегию
- **🔒 Криптографическая стойкость** - AES-256-GCM, ECDH, Perfect Forward Secrecy

## 🔧 Установка

### Требования
- Python 3.8 или выше
- Linux/Unix система с поддержкой TUN/TAP
- Root/Administrator права (для настройки сетевых интерфейсов)

### Быстрый старт

# Установка зависимостей
pip install -r requirements.txt

# Запуск в режиме клиента
sudo python -m anemone.client --profile stealth --server vpn.example.com

# Запуск сервера
sudo python -m anemone.server --port 443 --profile youtube

Использование Docker
# Сборка образа
docker build -t anemone-vpn .

# Запуск контейнера
docker run --cap-add=NET_ADMIN --device=/dev/net/tun -it anemone-vpn

📖 Документация
Основные команды CLI

# Запуск с автоматическим выбором профиля
anemone start --server your.server.com

# Просмотр статуса
anemone status

# Смена профиля маскировки
anemone profile set zoom

# Статистика соединения
anemone stats --detailed

# Ручное переключение транспорта
anemone transport switch quic

Конфигурация
Создайте файл ~/.anemone/config.yaml:

server: vpn.anemone.net
port: 443
profile: adaptive  # adaptive, youtube, zoom, web, telegram
transport: auto    # auto, webrtc, quic, websocket
encryption:
  algorithm: AES-256-GCM
  key_rotation: 3600  # секунды
fragmentation:
  enabled: true
  min_size: 50
  max_size: 1400
  ttl_mask: true
content_filler:
  enabled: true
  sources:
    - rss: http://feeds.bbci.co.uk/news/rss.xml
    - api: https://api.telegram.org/...
